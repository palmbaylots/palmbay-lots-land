3. YOU MISDIAGNOSED THE GSC "DUPLICATE WITHOUT USER-SELECTED CANONICAL" WARNING.

This warning is informational, not a penalty. It happens on React/Vue/Next single-page apps because the server returns 200 OK for every URL — even non-existent ones — so Google sees several URLs returning the same React shell and treats them as duplicates.

The correct fix is NOT to mess with canonicals or domain redirects. The correct fix is:

a) Add a catch-all <Route path="*"> that renders a 404 component with <meta name="robots" content="noindex, follow"> and a self-referencing canonical via react-helmet-async.

b) Add <Helmet> blocks with noindex to any conditional "Not Found" states inside dynamic routes (e.g., PropertyDetail.jsx and BlogPost.jsx when useParams() returns a slug that does not exist in the database).

c) Remove any static <meta name="robots"> from index.html so per-page robots tags from react-helmet-async are not duplicated.
